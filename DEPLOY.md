# Nexus Tax Engine — Deployment Guide

## Quick Start (local)

```bash
cd nexus-tax-engine
npm install
cp .env.example .env      # edit with your values
npm run dev               # dev mode with hot reload
# or
npm run build && npm start # production build
```

Server runs at http://localhost:3000

---

## Deploy to Render (recommended)

### Step 1 — Push to GitHub

```bash
git init
git add .
git commit -m "Initial commit: Nexus Tax Engine"
gh repo create nexus-tax-engine --public --push --source .
# or push to an existing repo
```

### Step 2 — Create Render service

1. Go to https://render.com and sign in
2. Click **New → Web Service**
3. Connect your GitHub repo
4. Render auto-detects `render.yaml` — confirm the settings:
   - **Build:** `npm install && npm run build`
   - **Start:** `npm start`
   - **Health check:** `/health`
5. Click **Deploy**

### Step 3 — Add a custom domain

1. In Render dashboard → your service → **Settings → Custom Domains**
2. Add your domain (e.g. `api.yourdomain.com`)
3. Render provides the DNS record (CNAME) to add at your registrar
4. Buy a domain at Namecheap / Cloudflare Domains / Google Domains
5. Add the CNAME record and wait for DNS propagation (~5–30 min)

### Step 4 — Secure the API

Your `API_KEY` is auto-generated by Render. To find it:
- Render dashboard → your service → **Environment** tab
- Copy the `API_KEY` value

All requests (except `/health`) must include:
```
x-api-key: <your-api-key>
# or
Authorization: Bearer <your-api-key>
```

## API Reference

### Health
```
GET /health
```

### Tax Rates
```
GET  /rates                        — All 50 states + DC
GET  /rates/:state                 — e.g. /rates/CA?rateType=combined_avg
GET  /rates/no-tax-states          — States with no sales tax (AK, DE, MT, NH, OR)
```

### Nexus
```
GET  /nexus/thresholds             — All economic nexus thresholds
GET  /nexus/thresholds/:state      — e.g. /nexus/thresholds/NY

POST /nexus/check
{
  "stateAbbreviation": "CA",
  "totalSales": 600000,
  "totalTransactions": 300,
  "hasPhysicalPresence": false
}

POST /nexus/bulk-check
{
  "salesByState": {
    "CA": { "totalSales": 600000, "totalTransactions": 300 },
    "TX": { "totalSales": 450000, "totalTransactions": 150 }
  },
  "physicalPresenceStates": ["NY"]
}
```

### Calculate
```
POST /calculate
{
  "destinationState": "CA",
  "saleAmount": 199.99,
  "rateType": "combined_avg"   // state_only | combined_avg | combined_max
}

POST /calculate/bulk
{
  "transactions": [
    { "id": "order-001", "destinationState": "CA", "saleAmount": 199.99 },
    { "id": "order-002", "destinationState": "TX", "saleAmount": 49.95 }
  ]
}
```

## Rate Types

| rateType | What it uses |
|---|---|
| `state_only` | Base state rate only |
| `combined_avg` | State + average local rate (default, good for most cases) |
| `combined_max` | State + maximum local rate (conservative / worst-case) |

---

## Important Notes

- **Rates are current as of Q1 2025.** Update `src/data/stateTaxRates.ts` and `src/data/nexusThresholds.ts` periodically.
- For address-level precision, integrate a geocoding API (TaxJar, Avalara) — the current engine uses state + average local rates.
- Always consult a tax professional for production compliance obligations.
